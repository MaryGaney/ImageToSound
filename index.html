Baseline Comparisons

Three baselines show that structured DSP mappings produce more meaningful outputs than degenerate alternatives.

B1

Brightness → amplitude

Maps overall brightness to a single sine wave amplitude. No spatial or spectral structure - every image sounds like a simple volume-varying tone. Select in the dropdown above.

B2

Random tones

Generates random frequencies and amplitudes uncorrelated with the image. Any perceptible structure in the full system must come from the image, not chance.

B3

Fixed harmonics

Synthesizes a fixed harmonic series] regardless of image content. Demonstrates that timbre variation in the full system comes from image features.

About this project

Course connection

Every DSP step in this system maps directly to a homework assignment. Sampling & quantization, convolution & FIR filtering, Fourier synthesis, and ideal sinc filters are all present as identifiable stages.

No machine learning

All mappings are deterministic and hand-designed. Given the same image and the same parameters, the system always produces the same audio. There are no trained models, embeddings, or neural networks involved.

Implementation

The synthesis engine (dsp.js) is plain JavaScript that mirrors the Python DSP code from the course: the same mid-tread quantizer, the same sinc-based FIR kernels, and the same phasor-sum harmonic synthesis - running in the browser at Fs = 44 100 Hz via Web Audio API.

Limitations & future work

The current system processes images column-by-column (left-to-right scan). Future improvements could include 2D spatial feature extraction, perceptual frequency scaling, and envelope shaping per-note to reduce clicks at segment boundaries.
